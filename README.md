## Описание

**ИИ-ассистент по ведению деловых переговоров** — это Telegram-бот, разработанный в VS Code с использованием современного стека:
- **OpenRouter API** — унифицированный доступ к лучшим LLM-моделям (включая OpenAI)
- **LangChain** — для контекстной обработки, управления памятью и логикой ответов
- **Собственная база знаний (FAQ)** — структурированные карточки решений по ключевым бизнес-сценариям

Бот помогает менеджерам, предпринимателям и консультантам:
- Быстро находить аргументы для встреч с клиентами, инвесторами и ЛПР
- Выявлять скрытую «бизнес-боль» в запросах
- Структурировать мысли и доносить ценность уверенно и ясно

## Ключевые возможности

### Интеллектуальный диалог
- Распознавание намерения пользователя и контекста запроса
- Работа с возражениями по схеме: **Признание → Уточнение → Аргумент → Пример → CTA**
- Естественная подача ответа в стиле опытного консультанта — без «роботизированности»

### Анализ бизнес-проблем
Поддержка сценариев в сферах:
- Продажи и переговоры
- Маркетинг и конверсия
- Операционная эффективность
- Финансы и ROI
- Масштабирование и автоматизация

Выявляет скрытые проблемы:  
*низкая скорость, плохие данные, ошибки в процессах, сопротивление команды и др.*

### Интеграция с базой знаний (FAQ)
Каждая карточка содержит:
- **Контекст** — в какой ситуации применима
- **Боль** — какую проблему решает
- **Решение** — что предлагает
- **KPI** — какие метрики улучшает
- **CTA** — призыв к действию
- **Minicase** — короткий пример из практики

Бот комбинирует до 2 карточек в единый, связный и убедительный ответ.

## Этапы реализации

### 1️⃣ Формирование базы знаний (FAQ)
- Сбор карточек с аргументами, кейсами и техниками  
- Группировка по контекстам: продажи, маркетинг, операции и др.  
- Определение правил выбора релевантных карточек  
- Построение схемы `auto-logic` для динамического подбора контента

### 2️⃣ Проектирование системного промпта
Промпт включает:
- Чётко определённую **роль** и **цели**  
- **Правила анализа** входящего запроса  
- Стандартизированную **структуру ответа**  
- **Стиль консультанта**: уверенный, ясный, ориентированный на результат (ROI)  
- Стратегии **работы с возражениями**  
- Требования к лаконичности и убедительности формулировок

### 3️⃣ Построение логики в LangChain
- Создание `Chain` с поддержкой **Memory** для учёта контекста диалога  
- Интеграция правил и карточек из FAQ  
- Единообразное **форматирование ответов**  
- Управление **намерениями пользователя** (intent detection)

### 4️⃣ Взаимодействие через OpenRouter API
- Единая точка доступа к сотням моделей  
- Автоматический выбор оптимальной модели по стоимости и качеству  
- Поддержка стриминга и совместимость с OpenAI-SDK  

### 5️⃣ Интеграция с Telegram-ботом
- Обработка входящих сообщений через **webhooks** или **polling**  
- Контекстно-зависимая подача ответов с учётом истории диалога

### 6️⃣ Тестирование
- Валидация логики сопоставления FAQ и запросов  
- Проверка корректности реакций на возражения  
- Тонкая настройка стиля: корректировка уверенного тона
